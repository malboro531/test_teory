Локатор - это механизм для определения местонахождения элемента относительно HTML документа, над которым необходимо
выполнять какие-то действия.

Важным различием между CSS и XPath локаторами является то, что, используя XPath, мы можем производить перемещение как в
глубину DOM иерархии, так и возвращаться назад. Что же касается CSS, то тут мы можем двигаться только в глубину.
Это значит, что с XPath можем найти родительский элемент, по дочернему.

Атрибуты поиска:
1. By.id(String id) - Поиск элемента по атрибуту id
2. By.name(String name) - Поиск элемента по атрибуту name
3. By.className(String className) - Поиск элемента по атрибуту class
4. By.tagName(String tagName) - Поиск элемента по тегу
5. By.linkText(String linkText) - 	Поиск элемента по тексту ссылки
6. By.partialLinkText(String partialLinkText) - Поиск элемента по частичному тексту ссылки
7. By.cssSelector(String cssSelector) - Поиск элемента по CSS селектору
8. By.xPath(String xpath) - Поиск элемента по XPATH запросу

Оси XPath
Ось в XPath - это специальная концепция, которая используется для указания направления движения при поиске элементов в
документе. Оси позволяют выбирать элементы, которые относятся к определенным отношениям с другими элементами в дереве
XML или HTML.
В XPath есть несколько осей, которые можно использовать при создании выражений:

1) nav[@id='navbar']/ancestor::* - предки текущего элемента
2) nav[@id='navbar']/parent::* - родители текущего элемента
3) nav[@id='navbar']/descendant::* - потомки текущего элемента
4) nav[@id='navbar']/child::* - дочерние текущего элемента
5) nav[@id='navbar']/preceding::* - предшествующие текущему элементу
6) nav[@id='navbar']/following::* - последующие текущему элементу

Для поиска элементов в мобильных приложениях на платформах iOS и Android используются специфичные локаторы, аналогичные
селекторам или XPath веб-приложений. Вот основные типы локаторов для поиска элементов в мобильных приложениях:

1. ID (Идентификатор): Каждый элемент в мобильном приложении может иметь уникальный идентификатор. Это самый быстрый и
надежный способ поиска элементов, если такие идентификаторы заданы.
2. XPath: Вы можете использовать XPath-запросы для поиска элементов по их иерархическому расположению в приложении.
XPath может быть полезен для сложных запросов.
3. Accessibility ID (ID доступности): Это специальный атрибут, который разработчики могут назначить элементам для
облегчения их поиска. Он особенно полезен для тестирования на iOS.
4. Class Name (Имя класса): Элементы могут быть найдены по имени их класса. Это может быть полезно, если элементы имеют
одинаковый класс.
5. Name (Имя): Вы можете искать элементы по их текстовому имени или метке.
6. Link Text и Partial Link Text (Текст ссылки и Частичный текст ссылки): Для поиска текстовых элементов, таких как
ссылки.
7. XPath с использованием ресурсов Android (для Android): В Android можно использовать ресурсы, такие как "resource-id",
для поиска элементов.
8. UIAutomator (для Android): В Android также существует фреймворк UIAutomator, который позволяет выполнять более
сложные действия и поиск элементов.
9. Predicates (Предикаты) (для iOS): В iOS можно использовать предикаты для выполнения более сложных запросов.
