Kanban – метод управления разработкой, при котором задачи распределяются равномерно между всеми участниками
команды разработки, реализует принцип «точно в срок», и ограничивается одновременное максимальное количество
выполняемых задач.

Основные принципы:
    - Визуализация – доска с карточками-задачами (user-stories в разработке продукта).

    - Имеется план разработки, отсортированный по приоритету (backlog в разработке продукта).
    Может меняться в любой момент.

    - Ограничение одновременно выполняемых задач. Постоянная оптимизация процессов.
