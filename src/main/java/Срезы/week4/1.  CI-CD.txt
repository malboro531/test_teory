CI/CD - подход, в котором мы выстраиваем процесс разработки программы так, чтобы можно было регулярно заливать свой код
с новыми фичами в проект, который ты делаешь, а затем все это дело должно автоматически собираться, тестироваться и
равзертываться.

Continuous Integration (непрерывная интеграция) - это автоматизированный процесс включающий в себя сборку,
развертывание и тестирование ПО без участия человека.

Этапы CI:
    - Написание кода. Разработчики пишут код своего модуля и проводят тестирование в ручном режиме.
    После этого результат работы соединяется в главной ветке с текущей версией проекта.
    После того, как в главной ветке публикуются весь код модулей, начинается второй этап.

    - Сборка. Выбранная система контроля версий инициирует автоматическую сборку и последующее тестирование
    проекта. Триггеры для активации сборки могут быть настроены самостоятельно.
    Для автоматизации сборки применяется Jenkins или другой инструмент.

    - Ручное тестирование. После проверки CI-системой работоспособности тестовой версии код передается для ручного
    исследования.

Continuous Delivery (непрерывная доставка) - разворачивание протестированной части нашего ПО
Тут понимается связь отдельных частей кода приложения между собой в единый продукт и доставка обновлений конечному
пользователю

Continuous Deployment (непрерывная поставка) - автоматическое разворачивание протестированной части нашего ПО в
продуктивную среду
