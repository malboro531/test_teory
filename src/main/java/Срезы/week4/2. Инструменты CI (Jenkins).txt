Jenkins — это Java-приложение.
У него есть контроллер или Master Mode — управляющий центр, который занимается планированием задач.
Он запускает задачи согласно установленному расписанию на слэйвах, которые вы к нему прикрепили.
Помимо этого контроллер хранит логи задач. Вся история хранится только на Master Mode.

Слэйвы или агенты — это то, что непосредственно выполняет сами задания, вспомогательные серверы.
Контроллер запускает задачу и говорит агенту выполнить её, агент выполняет задачу и возвращает результат
контроллеру. Контроллер получает результат и сохраняет его в build-логе.

Порядок установки:
    1. установить Java
    2. скачать файл и запустить установку
    3. переходим на localHost:8080
    4. вводим пароль, ссылка на него указана над окном ввода пароля
    5. установка плагинов, варианты
        - выбрать по умолчанию
        - выбрать свои
    6. создаем first admin user -> start using jenkins
    7. появляется рабоче пространство
        - new item (виды: job, folder, pipeline)
        - people
        - build history
        - manage jenkins
        - my views
        - credentials
        - localable recourses
        - new view
    8. жмем new item -> вводим название -> freestyle -> freestyle project
    9. далее идут настройки
        - source code manager -> git -> repository URL -> прописываем данные для доступа к репо ->
        можно установить build triggers что бы Jenkins реагировал на события, напр при обновлении репо
        - build -> ставим галочку около поля invoke Gradle -> Gradle version = default ->
        выбираем таску build model, напр из проекта на Gradle -> можем добавить build steps для добавления шагов
    10. настраиваем Jenkins в проекте
