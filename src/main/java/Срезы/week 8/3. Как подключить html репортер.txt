1. Добавление зависимости:

<dependencies>
    <!-- Другие зависимости -->
    <dependency>
        <groupId>io.qameta.allure</groupId>
        <artifactId>allure-testng</artifactId> <!-- Используйте allure-junit5, если используете JUnit 5 -->
        <version>2.14.0</version> <!-- Укажите актуальную версию Allure -->
    </dependency>
</dependencies>

2. Настройка Allure:
Создайте файл allure.properties в каталоге src/test/resources вашего проекта и укажите в нем путь для сохранения отчетов:

allure.results.directory=target/allure-results

3. Использование аннотаций:
Для создания отчетов Allure в тестах используются аннотации. Вот несколько основных аннотаций и их использование:

@Test: Аннотация для обозначения метода как тестового.

@Step: Аннотация, которая позволяет определить шаг тестового сценария. Применяется к методам и используется для создания
структуры отчета.

@Feature: Аннотация для указания функциональности (фичи), которую тестирует данный сценарий.

@Story: Аннотация для указания истории, связанной с тестовым сценарием.

@Issue("BUG-12345"): Аннотация для связи с багом

@TmsLink("TEST-54321"): Аннотация для связи с задачей

@Severity(SeverityLevel.NORMAL): Аннотация для серьезности бага

@Description("Test case description goes here."): Аннотация для описания

@Link(name = "Issue Tracker", url = "https://example.com/issue-tracker/BUG-12345"): Аннотация для добавления гиперссылок

4. Генерация отчетов:
Выполните в командной строке команду для генерации отчета Allure:

allure generate target/allure-results --clean

Отчет будет создан в каталоге target/allure-report. Вы можете открыть его в браузере для просмотра результатов.

5. Запуск тестов:
Запустите ваши тесты как обычно, используя вашу тестовую среду или систему сборки.

6. Просмотр отчетов:
Отчеты Allure можно просматривать в интерактивном веб-интерфейсе, который позволяет анализировать результаты тестов и
делать выводы о прохождении тестовых сценариев.
