Оператор UNION в SQL используется для объединения результатов двух или более запросов в один результат. Он устраняет
дубликаты строк из объединенных результатов. Каждый запрос должен возвращать тот же набор столбцов в одном и том же
порядке.

SELECT column1, column2, ...
FROM table1
WHERE condition
UNION
SELECT column1, column2, ...
FROM table2
WHERE condition;

Главные характеристики оператора UNION:

- Он объединяет результаты двух или более SELECT запросов.
- Количество столбцов и их типы во всех запросах должны совпадать.
- Результат будет содержать уникальные строки, то есть дубликаты будут автоматически удалены.
- Запросы в операторе UNION могут иметь разные условия (WHERE), но структура и количество столбцов должны быть
одинаковыми.
- Результат будет упорядочен в соответствии с первым запросом в операторе UNION.

Пример использования оператора UNION:

Допустим, у нас есть две таблицы: customers и vendors, и мы хотим получить список уникальных имен клиентов и
поставщиков:

SELECT customer_name FROM customers
UNION
SELECT vendor_name FROM vendors;

Этот запрос вернет список уникальных имен как клиентов, так и поставщиков из обеих таблиц.

Вы также можете использовать оператор UNION с более чем двумя запросами, если необходимо объединить результаты из
большего количества таблиц или условий.
